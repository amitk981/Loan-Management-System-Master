## Scope

- Replace `@object_scope_cases(...)` metadata discovery with one executable row runner/result ledger.
  A row may enter completeness only after it has evaluated the six-field action, invoked the matching
  public write on the same persisted out-of-scope resource, asserted exact disabled reason and
  `OBJECT_ACCESS_DENIED`, and compared the complete before/after evidence snapshot.
- Cover eligibility run, loan-limit calculate, appraisal create/update/revalidate/submit-review/
  review, and sanction submit. Keep each public write and projector behind its production module
  interface; do not call private helpers to manufacture coverage.
- Make false completeness mechanically fail: deleting the write call, projection assertion, denial
  category assertion, or evidence comparison from any row must remove/fail that row rather than leave
  its action code advertised.
- Preserve the existing HTTP `403` non-disclosure regressions and all role, permission,
  maker-checker, state, provenance, rejection-payload, immutable-review, stale-state, and PostgreSQL
  race coverage. Do not change business rules or production behavior to satisfy the ledger.

- Mutation tests or deliberately incomplete fixture rows prove that omitting each required phase
  produces a focused completeness failure.
- All eight rows retain exact six-field shape/reason/category and zero persisted evidence delta;
  focused eligibility/limit/appraisal/sanction HTTP object denials remain standard `403` envelopes.
